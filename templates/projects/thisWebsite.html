Network configuration

Now the network configuration. As I stated before, I built this website from the very scratch, that includes configuring my home network allowing internet traffic. Luckily that wasn't hard, since I already had knowledge and experience on CISCO networking and knew the steps that had to be done to deploy a webserver. The first thing that had to be done is giving the hosting machine a static IP so the PAT (Port Address Translation) would continue working even if the machine disconnected and reconnected from the network. For that you must go to the webpage on your network gateway by simply typing your gateway IP on any web browser, which is normally either 192.168.1.1 or 192.168.1.254 (unless you are working with CISCO equipment, then you must configure it on the terminal), introducing the credentials and look for the DHCP (Dynamic host configuration protocol) section. Once there I simply searched the hosting device MAC Address and set the static IP to be 192.168.1.1.
The next step was look for the section to configure the PAT, in home network modems it is usually called «port translation», «port mapping» or something like that. This protocol allows us to connect on a private network from the internet by allowing incoming traffic from a especified port, which in this case was port 80. The following picture shows the record that contains this configuration, being the first value an identifying label, the second one the local inside IP (which is static), the third one the allowed protocol traffic, the fourth one the local inside port or the hosting machine port where the server is running and finally the local outside port (which is the port where incoming traffic is allowed).
